# Monitor Server

## Background

This repository is developed to track the progress of the scripts that I deployed in my home server. The scripts that I write usually take a while to finish, e.g. web scraping or sending tasks to AI. During this time, I can only pray and hope that the scripts can do well. Therefore, instead of just praying and hoping, I built a simple monitoring server inspired from my work to track the progress of the script.

## Requirement(s)

- Python
- Redis server (hosted in my home server with Ubuntu Server as the OS)
- uv (for easy python environment setup)

## How to start

1. Create an ".env" file in the root

    ```.env
    REDIS_HOST=[Your Redis Server IP Address]
    REDIS_PORT=[Your Redis Server Host]
    REDIS_PASSWORD=[Your Redis Server Password]
    ```

2. Create a `scripts` folder
3. Create a simple "Keep Alive" script

    ```py
    from time import sleep
    from src.classes import Logger

    # the ID is generated by running `uuid.uuid4().hex`
    # the ID follows a very strict format: must be from `uuid.uuid4().hex`
    # once ID initialised and saved in redis database, clearing/modifying it will
    # cause different events being saved
    ID='0012773401804e1aa1dbd57077b51582' # important!
    NAME='Keep Alive Script' # important!
    DESCRIPTION='Sending Keep Alive events' # important!

    def execute():
        
        # this object's job is to send the log to the database
        logger = Logger(ID)
        
        # sending keep alive
        while True:
            logger.log("I'm alive.")
            sleep(2)
    ```

4. Run the `Flask` app: `uv run flask --app src run --debug`
